# tokmd 1.13

tokmd 1.13 is the syntax-aware evidence packet release. It extends the Bun UB
evidence path from 1.12 with a manifest-first packet, optional syntax receipts,
and advisory review priority that can point reviewers, bots, and agents at the
highest-risk seams first.

Use this page as the user-facing release note. For the lane-by-lane maintainer
record, see the [1.13 release ledger](1.13-ledger.md). For dogfood evidence,
see the [1.13 syntax evidence dogfood note](1.13-dogfood.md).

This release does not make tokmd a verifier. Syntax receipts and review
priority are advisory parser evidence. They do not prove reachability, undefined
behavior, safety, correctness, CI proof, or merge readiness.

## Review Bot Artifacts

The intended packet layout is:

```text
sensors/tokmd/
  manifest.json
  analyze.md
  analyze.json
  context.md
  syntax.json
```

Generate the packet from explicit refs and changed paths:

```bash
BASE="${BASE:-origin/main}"
HEAD="${HEAD:-HEAD}"

mkdir -p sensors/tokmd
rm -f sensors/tokmd/syntax.json

tokmd analyze \
  --preset bun-ub \
  --format md \
  --effort-base-ref "$BASE" \
  --effort-head-ref "$HEAD" \
  --no-progress \
  "$@" \
  > sensors/tokmd/analyze.md

tokmd analyze \
  --preset bun-ub \
  --format json \
  --effort-base-ref "$BASE" \
  --effort-head-ref "$HEAD" \
  --no-progress \
  "$@" \
  > sensors/tokmd/analyze.json

tokmd context \
  --budget 64000 \
  "$@" \
  > sensors/tokmd/context.md

if tokmd syntax --help >/dev/null 2>&1; then
  tokmd syntax \
    --no-progress \
    "$@" \
    > sensors/tokmd/syntax.json
fi

tokmd evidence-packet \
  --preset bun-ub \
  --base "$BASE" \
  --head "$HEAD" \
  "$@"
```

Open `sensors/tokmd/manifest.json` first. It records packet status, artifact
paths, warnings, errors, non-claims, reproduction commands, and advisory
`review_priority` entries. Open `syntax.json` only as parser-backed supporting
evidence for those priority entries.

## Highlights

- `tokmd evidence-packet` writes a versioned `sensors/tokmd/manifest.json`
  contract over `analyze.md`, `analyze.json`, `context.md`, and optional
  `syntax.json`.
- `tokmd syntax` emits feature-gated `tokmd.syntax_receipts.v1` packets for
  Rust, TypeScript, TSX, and Python when the binary is built with the `ast`
  feature.
- Syntax receipts record parser capability, parse status, syntax facts,
  imports, exports, call sites, risk seams, review signals, warnings, errors,
  and non-claims.
- Evidence packet manifests index optional syntax artifacts and degrade to
  `partial` when syntax evidence is malformed, failed, missing by explicit
  request, or otherwise advisory.
- Syntax review signals feed advisory `review_priority` entries with refs back
  into `syntax.json`.
- Cockpit and handoff surfaces can point reviewers and agents to the packet
  artifact family and regeneration commands.
- Complexity reporting now treats average cyclomatic complexity as an average
  across functions, rather than comparing a per-file total against max
  cyclomatic complexity.
- Dogfood runs prove the current packet flow on one real tokmd Rust source file
  and representative Rust, TypeScript, and Python syntax samples.

## For Reviewers

Use `manifest.json` as the first file. The status tells you whether the packet
is complete, partial, or failed. The `review_priority` array is a reading order,
not a verdict.

For example, a syntax-backed packet can prioritize:

- Rust panic, unwrap, expect, indexing, allocation, and FFI seams;
- TypeScript or TSX native-boundary calls, `bun:ffi`, dynamic imports, and
  exported public surfaces;
- Python `eval`, dynamic imports, subprocess, file I/O, native/FFI hints, and
  exception seams.

Open the referenced syntax receipt entries before making any claim about a
file. Syntax evidence can reduce search space, but it does not prove public
reachability or guard adequacy.

## For Bot Operators

Treat packet status strictly:

- `complete`: required analysis and context artifacts are present and valid.
- `partial`: evidence is usable only with the named warnings and limits.
- `failed`: do not attach as valid review evidence.

If the installed binary does not include `tokmd syntax`, omit `syntax.json`.
`tokmd evidence-packet` remains valid over the required analysis and context
artifacts, but syntax-backed `review_priority` entries will be absent.

On Windows PowerShell, avoid plain `>` redirects for JSON artifacts unless the
host is known to write UTF-8. Prefer tokmd output flags where available or an
explicit UTF-8 writer. `tokmd evidence-packet` rejects non-UTF-8 JSON instead
of silently indexing it.

## What Did Not Change

- No UB-found or UB-absent claim.
- No public reachability proof.
- No semantic call graph or type resolution.
- No default AST-backed `analyze`, `cockpit`, `context`, handoff, FFI, Python,
  Node, WASM, or browser output.
- No public `tokmd review` command.
- No proof promotion.
- No default Codecov upload.
- No evidencebus runtime integration.
- No release, publish, signing, tag, crate, image, or alias mutation from
  `tokmd-swarm`.

## Deferred

Full public-reachable panic-seam receipts remain post-1.13 unless explicitly
assigned. The current release can surface advisory panic-like syntax signals,
but a dedicated receipt still needs its own schema, source hints, guard evidence,
confidence model, category diversity, and non-claims.
