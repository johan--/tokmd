# tokmd 1.13 Release Ledger

## Purpose

tokmd 1.13 is the syntax-aware evidence packet release. It turns the 1.12 Bun
UB evidence path into a manifest-first packet that can optionally include
parser-backed syntax evidence and advisory review priority.

This ledger is the maintainer release record. It is intentionally narrow: the
1.13 cycle does not add a public reachability proof, UB detector, semantic call
graph, default AST behavior, or release mutation from `tokmd-swarm`.

## 1.13.1 Correction Record

`v1.13.0` was published and installable, but the default crate install did not
include the `ast` feature. That made the documented `tokmd syntax` command
unavailable from `cargo install tokmd --version 1.13.0`, even though the syntax
command was present in all-feature development and CI builds. The `v1.13.1`
correction makes `ast` part of the default `tokmd` feature set so the installed
binary matches the documented syntax-aware evidence packet workflow.

The same correction also closes the remaining capped-scan trust gap for
complexity evidence. File-backed complexity analysis has a default per-file
content window; when eligible source exceeds that window, the analysis receipt
now carries `complexity scan bounded` warnings and `partial` status. This keeps
review packets from treating clipped complexity input as complete evidence.

Correction proof included:

- local default install smoke from the release-correction branch;
- `tokmd syntax --help` and `tokmd evidence-packet --help` from the installed
  binary;
- a capped Rust complexity smoke producing `status=partial` and bounded
  warnings;
- focused CLI and analysis regressions;
- affected proof over analysis complexity, analysis orchestration, CLI, and
  workspace dependency graph scopes;
- green `tokmd-swarm` PR #235 and publication import PR #2564.

Representative PRs are examples, not an exhaustive list. The release work lands
as narrow swarm PRs, then moves into the publication repository by merge commit
before tagging.

## Evidence Contract

The 1.13 evidence packet is:

```text
sensors/tokmd/manifest.json
sensors/tokmd/analyze.md
sensors/tokmd/analyze.json
sensors/tokmd/context.md
sensors/tokmd/syntax.json
```

`syntax.json` is optional unless the workflow explicitly requires it. When it is
present, `tokmd evidence-packet` indexes it and may derive advisory
`review_priority` rows from its `review_signals`.

The packet must be:

- scoped to the changed paths passed to the command;
- explicit about whether `BASE` and `HEAD` resolve;
- readable enough for a human reviewer to inspect first;
- stable enough for bot and ledger ingestion;
- clear about included, truncated, skipped, missing, malformed, failed, or
  partial evidence;
- bounded by non-claims that prevent syntax evidence from becoming a verifier
  claim.

## Validation Pattern

The release lane used focused regressions first, then release-facing checks:

- syntax parser registry tests;
- language-specific Rust, TypeScript, TSX, and Python syntax fact tests;
- cross-language review signal normalization tests;
- CLI syntax integration tests for scoped output, advisory partial status, and
  missing inputs;
- evidence packet tests for optional syntax artifacts, warning preservation,
  review-priority ranking, failed syntax, malformed syntax, and explicit missing
  syntax artifacts;
- cockpit and handoff integration tests for packet artifact links;
- dogfood packet runs under `target/dogfood/v1.13-syntax/`;
- docs, doc-artifacts, proof-policy, affected routing, publish-surface, and
  hosted PR checks for the changed surfaces.

Before tagging, run the release-readiness checks:

```bash
cargo fmt --all -- --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test --workspace --all-features
cargo xtask version-consistency
cargo xtask publish-surface --json --verify-publish
cargo xtask doc-artifacts --check
cargo xtask docs --check
cargo xtask proof-policy --check
cargo xtask affected --base origin/main --head HEAD --json-output target/proof/affected-release.json
cargo xtask proof --profile affected --base origin/main --head HEAD --plan --plan-json target/proof/proof-plan-release.json --evidence-json target/proof/proof-evidence-release.json
```

## Dogfood Findings

The dogfood report found:

- complete manifests with zero warnings and zero errors for four scoped packet
  runs;
- useful first-read ordering for Rust panic/native seams, TypeScript native
  boundary calls, and Python dynamic/native seams;
- a Windows PowerShell redirect encoding friction where `>` produced non-UTF-8
  JSON and `tokmd evidence-packet` correctly rejected it;
- ranking noise in real tokmd Rust parser source where unit-test assertions
  crowded the top priority entries;
- category crowding where native-boundary hints could push subprocess, dynamic
  import, file I/O, or exception seams out of the capped top 10.

## Deferred

1.13 does not add:

- a UB detector;
- public reachability proof;
- full public-reachable panic-seam receipts;
- semantic call graph or type resolution;
- public `tokmd review`;
- evidencebus runtime integration;
- proof promotion;
- default Codecov upload;
- default AST-backed public behavior;
- full-matrix migration;
- release, publish, signing, tag, crate, image, or alias mutation from
  `tokmd-swarm`.

Full panic-seam receipts remain a later schema-reviewed lane. The likely next
contract should include explicit source hints, guard evidence, confidence,
category diversity, and non-claims.
