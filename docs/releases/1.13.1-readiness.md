# tokmd 1.13.1 Release Readiness Report

Date: 2026-06-11

## Purpose

`v1.13.1` is a correction release for the 1.13 syntax-aware evidence packet
surface. `v1.13.0` was published, but a default crate install did not expose
`tokmd syntax`. This report records the correction proof before tagging
`v1.13.1`.

## Correction Scope

Included:

- default `tokmd` installs include the `ast` feature, making `tokmd syntax`
  available from the published crate without custom feature flags;
- bounded/default-capped file-backed complexity scans emit explicit
  `complexity scan bounded` warnings;
- bounded complexity scans mark the analysis receipt `partial`;
- CLI help and generated reference docs state the default file-backed scan cap.

Not included:

- no new syntax analyzer;
- no schema change to evidence packet manifests;
- no UB detection, reachability proof, safety proof, or merge-readiness claim;
- no release mutation from `tokmd-swarm`.

## Publication Flow

- `tokmd-swarm` PR #235 landed the correction and passed hosted CI after
  rerunning one canceled CPX42 routed job.
- `EffortlessMetrics/tokmd` PR #2564 imported that swarm state by merge commit.
- `tokmd-swarm/main` was fast-forwarded back to the publication merge commit.
- This release-prep branch is metadata-only after the publication import.

## Release Decision

This report began as the pre-tag readiness record for `v1.13.1`. The release
has since been published, and the post-release verification below records the
remaining distribution boundary.

## Post-Release Verification

After tagging `v1.13.1`, the release workflow completed successfully and
published the expected release artifact family. The installed-artifact smoke
verified:

- `cargo install tokmd --version 1.13.1 --locked`;
- `tokmd --version` reported `tokmd 1.13.1`;
- `tokmd syntax --help` was available from the installed binary;
- `tokmd evidence-packet --help` was available from the installed binary;
- a scoped evidence packet smoke produced a `complete` manifest.

Distribution caveat:

- the release workflow's Docker/GHCR publish job completed successfully;
- unauthenticated `docker manifest inspect ghcr.io/effortlessmetrics/tokmd:1.13.1`
  returned `denied`;
- the available GitHub token could not inspect the package API because it lacked
  `read:packages`.

This records GHCR public visibility as pending maintainer verification. It does
not invalidate the verified crates.io or GitHub release install paths, and it
must not be resolved by rewriting tags or rerunning release mutation without
maintainer approval.
